☁️ Why EC2 and Not EC²??

Created2024-11-11Updated2024-11-11

☁️ Your Local Computer in the Cloud ☁️

Amazon Elastic Compute Cloud (EC2) is one of the coolest web services in AWS! Imagine needing a computer right away – one that you can power up, install apps on, and even turn off when you are done. Amazon EC2 lets you do exactly that, but all in the cloud! Here is a breakdown of how it works and why it is a game-changer for developers, businesses, and learners alike.

What is Amazon EC2? 🖥️

Amazon EC2, or Elastic Compute Cloud, provides scalable and resizable virtual servers in the cloud. These virtual servers, called instances, can be started up or shut down on-demand to meet whatever computing needs you have – from hosting a website to running complex applications.

How Does Amazon EC2 Work? 🔧

  1. Choose Your Instance Type 💻
    Just like you would pick a computer based on its RAM and CPU power, you select an EC2 instance type based on what you need. EC2 offers a range of instances optimized for general use, computing, memory, and storage.

  2. Pick an Operating System 🐧
    EC2 lets you install different operating systems, like Linux, Windows, or even custom Amazon versions that are lightweight and efficient.

  3. Start and Stop Anytime ⏯️
    Launch an instance whenever you need it, and stop it when you are done to save on costs. AWS charges only for the computing time you use, so it’s super flexible.

  4. Manage and Secure 🔐
    EC2 provides tools to secure, monitor, and manage instances. You can control who accesses your instance and track its usage and performance.

Why Use Amazon EC2? 🌟

Amazon EC2 offers amazing advantages for developers and businesses alike:

  • Scalability: Need more power? Add more instances. EC2 scales with your needs.
  • Cost-Efficiency: Pay only for what you use. Save money by shutting down instances when they’re not needed.
  • Flexibility: Run any OS, application, or software you need.
  • Global Reach: Place your instances in AWS regions around the world for fast access and local compliance.

Key Features of Amazon EC2 ⚙️

  • Elastic Load Balancing: Balances traffic across multiple EC2 instances.
  • Auto Scaling: Automatically scales the number of instances based on traffic, so you always have the right amount of computing power.
  • Storage Options: Attach different types of storage, like Amazon EBS (Elastic Block Store), for reliable and flexible storage.

Getting Started with EC2 🚀

Ready to dive into Amazon EC2? Here is a simple way to get started:

  1. Sign in to AWS Console – Navigate to the EC2 Dashboard.
  2. Launch an Instance – Choose an instance type, OS, and any extra configurations.
  3. Access Your Instance – Connect to your instance via SSH for Linux or RDP for Windows.
  4. Experiment and Explore – Try installing applications or running scripts.

IAM WHO IAM🔒

Created2024-11-07Updated2024-11-10

What is IAM (Identity Access Management)?

IAM is a web service that helps securely control access to AWS resources. You can manage the resources that can be accessed and which actions can be performed.
You use IAM to control who is authenticated (signed in) and authorized (has permissions) to use resources. IAM provides the infrastructure necessary to control authentication and authorization for your AWS accounts.

Type of ways to Access to AWS Services 🚀

  1. Programmatic Access 🔑

    • For developers and apps that use AWS through code.
    • Uses access keys for security.
    • Can access AWS CLI, APIs, and SDKs.

  2. Console Access 💻

    • For people who log in to the AWS website.
    • Uses username and password.
    • You can enable MFA (Multi-Factor Authentication) for added security.

Types of Security Credentials 🔐

AWS uses various types of credentials to secure access:

  • Email and Password for the root user (admin).
  • Access Keys for programmatic access.
  • IAM Username and Password for AWS Console.
  • MFA adds an extra layer of security.

🔒 Best Practice: Is to use the root user to create an IAM user first. Then, securely lock away the root user credentials. Use these credentials only when you must perform service management tasks that you cannot accomplish in other ways.

Types of IAM Policies 📜

Policies are like “permission cards” 📇 that specify what actions are allowed or denied for users and resources. Here is a breakdown of the main types of IAM policies:

  1. Identity-Based Policies 👤

    • Attached to users, groups, or roles.
    • Types: Managed Policies (reusable) and Inline Policies (specific to one user).
    • Example: Granting Sally permission to read data in S3.

  2. Resource-Based Policies 📦

    • Attached directly to specific AWS resources (like an S3 bucket).
    • Controls who can access the resource.
    • Example: Allowing only trusted accounts to access a specific S3 bucket.

  3. Service Control Policies (SCPs) 🏢

    • Used in AWS Organizations to apply policies across accounts.
    • Acts as permission boundaries for groups of accounts.
    • Example: Allowing only specific services for a group of AWS accounts.

  4. Access Control Lists (ACLs) 📄

    • Simpler policies for specific services (like S3).
    • Controls which users can access a resource.
    • Unlike all other policies ACL does not use the JSON document structure.

Quick Summary 📝

  1. IAM is like a security guard for your AWS resources.
  2. Policies are rules that say who can access what.
  3. Permissions define the actions users can take.
  4. Access Types include programmatic access (CLI, SDK) and console access (AWS Console).

Remember: Avoiding using root user credentials for daily administration and use IAM roles to provide cross-account access. And implementing MFA to provide an additional level of account security.

☁️ Fun Fact

Amazon IAM was launched on June 30, 2011—five years after AWS started! Before IAM, all users in an AWS account had to share a single set of credentials, which wasn’t ideal for security. 🌥️

🤷🏽‍♂️What is Cloud Computing??☁️

Created2024-11-05Updated2024-11-07

Imagine a world where all your photos, files, and software do not need to live on a single device. Instead, they float in a space called the cloud, accessible from any internet-connected device. Sounds futuristic? What if I told you it isthe world we are presently living the world of cloud computing!

📌 What Exactly is Cloud Computing?

In simple and few words cloud computing is the delivery of computing/IT resources over the internet. While there areseveral ways to define cloud computing it all comes down to these five key aspects:

  1. Networking
  2. Data Management
  3. Storage
  4. Services
  5. Devices
    At its core, cloud computing is the delivery of the above and more—over the internet, or “the cloud.” Instead of owning and managing physical servers, companies rent flexible resources from cloud providers like Amazon Web Services (AWS), Microsoft Azure, and Google Cloud Platform (GCP).

The dynamic effectts of cloud adoption are already playing out and are most evident in the three wide-ranging and commoncloud computing model:
Software-as-a-Service(Saas)
Infrastructure-as-a-service(Iaas)
Platform-as-a-service(Paas)

Imagine a three-tiered pyramid with Iaas sitting at the base assisting system administrators followed by Paas in the middle aiding developers and service intergration and Saas sitting on top of our pyramid benefiting the end user.

There are different types of cloud deployment from our mani concern the public cloud to the traditional private cloud and lastly the hybrid cloud which is mostly left out in the cloud debate but I think the rise of it adds a whole different flavour.

🌍 The Future of the cloud 🌥️

The International Data Corporation(IDC) estimates that already in 2024 at least half of IT spending is cloud based and only set to grow over the coming years. In fact, it is more likely that virtually all entreprises worldwide will consume some form of cloud services, signalling that inevitably most applications and entreprise information flows will be cloud based. It will become more than just a consumption model it will be central to shapping business IT strategy.

☁️ FUN FACT

The term “cloud computing” originated from early network diagrams where the internet was often represented by a fluffy cloud shape. This abstract shape signified the complex infrastructure that was hidden behind the scenes. As companies began to host services remotely and over the internet, the term “cloud” symbolized the vast, intangible nature of this new technology.
